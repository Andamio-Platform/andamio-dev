# The model this course is built on

## Before we start

You can work through this course on the Andamio app or in an agent harness (Claude Code, Cursor). We'll help you set up either one.

The first lesson is about pedagogy, and that's deliberate. Senior judgment comes from doing the work yourself, especially when you don't have to. Every design decision in Andamio is downstream of that belief.

## The concept

Three ways to issue credentials. Each answers a different question.

| Model                     | What the credential proves                      | Examples                                                             |
| ------------------------- | ----------------------------------------------- | -------------------------------------------------------------------- |
| **Completion-centered**   | You showed up.                                  | LinkedIn Learning, Coursera, compliance training                     |
| **Test-gated**            | You passed a fixed exam.                        | AWS SA, CCNA, CompTIA A+                                             |
| **Contribution-centered** | You produced real work and a human accepted it. | A PR that ships, a script that runs on preprod, feedback on a lesson |

What a collaborator usually wants to know: *can this person do the work, and does someone I trust vouch for it?* Only the third model answers that directly. Andamio is built on it, and this course uses it with you from this first lesson.

## What this looks like on Andamio

| Shape               | How it works                                                                                                                  | Covered in |
| ------------------- | ----------------------------------------------------------------------------------------------------------------------------- | ---------- |
| **Course**          | Learner submits evidence, teacher accepts or asks for revision. Credential records who did the work and who vouched for it.   | Module 500 |
| **Project**         | Contributor does the work, manager accepts. Payment and credential settle in one transaction.                                 | Module 600 |
| **Lesson feedback** | Your critique of the course is itself evidence of contribution. The thing you're improving is the thing you're demonstrating. | Module 700 |

## Why on-chain

A database can track submissions. Two properties make on-chain settlement worth the complexity:

- **Portability.** The credential belongs to the learner, not the institution. It lives in their wallet, outlives the course that issued it, and can be verified without asking us.
- **Composability.** Any credential can become a prerequisite for another. A project can gate on a course credential. This only works if the prerequisite measures real capability — which is why the protocol requires contribution-centered credentials, not completion certificates.

1. **"Completed the 12-week Advanced Python bootcamp at Tech Academy."**
2. **"Passed the AWS Solutions Architect exam with a score of 92%."**
3. **"Submitted a working Cardano transaction script that built, signed, and landed on preprod. Reviewed and accepted by a course teacher."**

Write your answers before reading the rubric.

## Rubric

1. **Completion-centered** — names attendance, not capability.
2. **Test-gated** — names exam performance on fixed questions, not open-ended production.
3. **Contribution-centered** — names the artifact (a script that landed on preprod) and the human who accepted it.

The phrasing doesn't have to match. The reasoning does.
