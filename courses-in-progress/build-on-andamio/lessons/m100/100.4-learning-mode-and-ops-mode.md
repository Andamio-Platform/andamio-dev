# Learning Mode and Ops Mode

## Before we start

The `andamio-dev` plugin has two modes. You're in Learning Mode now — the default. Ops Mode unlocks progressively as you complete modules. This lesson explains the distinction and why the gate is shaped the way it is.

## Learning Mode

Learning Mode is the course you're reading. Seven modules, sequential, one SLT at a time. Every lesson has a self-check. Every module ends with an assignment. When a teacher accepts your assignment, that module's SLTs are marked complete in `progress.json` and the next module unlocks.

A developer who opens `andamio-dev` for the first time lands here.

## Ops Mode

Task-focused skills for getting work done quickly:

| Skill | What it does |
|-------|-------------|
| `/auth-setup` | Configure API keys and JWT auth for an environment |
| `/cli-guide` | Look up syntax for any `andamio` subcommand |
| `/explore-api` | Search the API spec by keyword |
| `/cost-estimator` | Calculate ADA cost of an operation or flow |
| `/troubleshoot` | Debug failed, stalled, or expired transactions |
| `/course-ops` | Run common course operations |
| `/project-ops` | Run common project operations |
| `/compound` | Capture session learnings into your knowledge base |

Each skill assumes you already know what the work is and gets out of your way.

## The gate

|---|---|

| M300 Reading Andamio | `/explore-api`, `/cli-guide`, `/cost-estimator` |

| M600 Projects End-to-End | `/project-ops` |
| M700 Build Something Real | `/compound` |

## Why the gate exists

A tool you don't understand is someone else's tool, even when you're the one holding it. If `/course-ops` creates a course and you don't know which transactions it built or which endpoints it called, you're dependent on the tool's author the moment something goes wrong. That's a black box you're renting, not a capability you own.

The gate turns that around. Every Ops skill waits behind a module that teaches what the skill wraps. When it unlocks, it's not a reward — it's an acknowledgment that handing you the tool would no longer be dishonest.

Same move as contribution-centered credentials: the credential proves the holder did the work. The Ops unlock proves the holder understands the tool.
